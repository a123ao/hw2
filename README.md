# 強化學習作業二：Q-learning 與 SARSA 於 Cliff Walking 之比較與分析

## 一、作業目的

本作業旨在實作兩種經典的強化學習演算法：**Q-learning**（離策略, Off-policy）與
**SARSA**（同策略, On-policy）。透過在經典的「懸崖行走（Cliff
Walking）」格子世界（Gridworld）環境中進行訓練，比較兩者在相同的環境與參數設定下的學習行為、收斂特性，以及最終學習到的策略差異。

## 二、環境描述與問題設定

本實驗採用 4 × 12 的矩形網格環境（Cliff Walking），詳細設定如下：

- **狀態空間（State Space）**：網格中的 48 個位置座標 `(row, col)`。
- **動作空間（Action Space）**：上、下、左、右（4 個離散動作）。
- **起點（Start）**：左下角 `(3, 0)`。
- **終點（Goal）**：右下角 `(3, 11)`。
- **懸崖區域（Cliff）**：底部區域 `(3, 1)` 到 `(3, 10)`。
- **獎勵機制（Reward）**：
  - 每走一步獲得 **−1** 的獎勵（鼓勵盡快到達終點）。
  - 掉入懸崖獲得 **−100** 的重大懲罰，並且會直接被傳送回起點。
  - 到達終點回合結束。

實驗的學習參數設定如下：

- **策略（Policy）**：$\epsilon$-greedy，其中 $\epsilon = 0.1$。
- **學習率（$\alpha$）**：$0.1$。
- **折扣因子（$\gamma$）**：$0.9$。
- **訓練回合數（Episodes）**：500 回合（為降低隨機性，實作中重複執行 50
  次並取平均）。

## 三、演算法實作說明

（完整實作程式碼請參考隨附的 `cliff_walking.py` 檔案。）

### 1. Q-learning (Off-policy)

Q-learning 的更新公式如下：
$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[ R_{t+1} + \gamma \max_{a} Q(S_{t+1}, a) - Q(S_t, A_t) \right]$$

在實作中，當我們身處狀態 $S_t$ 並採取動作 $A_t$ 抵達 $S_{t+1}$
後，**無論我們下一步實際會根據 $\epsilon$-greedy 採取什麼動作**，Q-learning
都是假設我們在 $S_{t+1}$ 會採取「價值最高的動作（貪婪動作）」來計算 TD
Target（即公式中的
$\max_a Q(S_{t+1}, a)$）。這使得它成為**離策略（Off-policy）**的方法，因為它評估的策略（貪婪策略）與收集資料時使用的策略（$\epsilon$-greedy）不同。

### 2. SARSA (On-policy)

SARSA 的更新公式如下：
$$Q(S_t, A_t) \leftarrow Q(S_t, A_t) + \alpha \left[ R_{t+1} + \gamma Q(S_{t+1}, A_{t+1}) - Q(S_t, A_t) \right]$$

在實作中，SARSA 會在抵達 $S_{t+1}$ 後，**先根據目前的 $\epsilon$-greedy
策略決定下一個動作 $A_{t+1}$**，然後使用這個實際將要執行的動作的 Q
值來更新當前的
$Q(S_t, A_t)$。這使得它成為**同策略（On-policy）**的方法，因為它評估並改善的策略與實際與環境互動的策略完全相同。

## 四、結果分析

### 1. 學習表現與收斂速度

根據典型的實驗結果與理論預期：

- **SARSA 收斂較快且平均獎勵較高**：在訓練的早期，SARSA 的累積獎勵曲線會比
  Q-learning 更快上升，並穩定在一個較高的數值（約 -25
  左右）。因為它在探索時考慮到了 $\epsilon$
  帶來的隨機掉崖風險，因此學會了遠離懸崖，減少了獲得 -100 懲罰的次數。
- **Q-learning 平均獎勵較低且波動大**：Q-learning
  的曲線通常會維持在較低的水準（約 -40 到 -50
  之間），因為它雖然學到了理論上最短的路徑，但在實際執行 $\epsilon$-greedy
  策略時，那 10% 的隨機動作很容易讓它掉入懸崖，導致過程中吃下大量懲罰。

### 2. 策略行為分析 (路徑視覺化)

如果我們將 $\epsilon$ 設為 0，測試兩種演算法最終學到的純貪婪策略路徑：

- **Q-learning 學習到的路徑（冒險且最佳化）**：
  - **路徑**：起點 -> 向上一步 -> 一路向右沿著懸崖邊緣走 -> 向下一步到達終點。
  - **行為分析**：Q-learning
    是在學習「最優策略（最短路徑）」。即使在訓練時經常因為隨機探索掉入懸崖，它更新
    Q 值時仍只看 $\max Q$，因此它「忽略」了探索帶來的風險，傾向於極端**冒險**。

- **SARSA 學習到的路徑（保守且安全）**：
  - **路徑**：起點 -> 向上走到最頂部（或遠離懸崖的安全距離） ->
    向右走到終點正上方 -> 向下走到終點。
  - **行為分析**：SARSA 學習的是「在 $\epsilon$-greedy
    策略下的最優路徑」。因為它在更新時考慮到了未來實際要採取的動作，它知道只要靠近懸崖就有
    10% 的機率會掉下去（因為
    $\epsilon$-greedy）。因此，為了最大化整體的期望獎勵，它選擇了繞遠路來迴避懸崖，傾向於**保守與安全**。

### 3. 穩定性分析

- **SARSA** 的學習過程非常穩定，在找到安全路徑後，每個 Episode
  的獎勵變動幅度小，因為遠離懸崖大幅降低了隨機掉落的意外。
- **Q-learning** 由於緊貼懸崖邊緣，只要 $\epsilon$-greedy
  觸發了向下的隨機動作，就會立刻掉入懸崖。這導致了其在學習過程中的單回合總獎勵呈現出極大的波動（忽高忽低）。
- **探索（Exploration）的影響**：探索率 $\epsilon$ 是造成兩者表現差異的核心。若
  $\epsilon \to 0$ 逐漸退火（Decay），SARSA 的路徑最終也會漸漸逼近 Q-learning
  的最短路徑；但若 $\epsilon$ 固定為 0.1，SARSA 就會永遠保持保守。

## 五、理論比較與討論

這兩種演算法的根本差異在於**TD Target 的計算方式**：

- **Q-learning (Off-policy)**：
  其更新基於「下一狀態的最佳可能行動」。它具備了分離「行為策略（Behavior
  Policy，用於探索）」與「目標策略（Target
  Policy，正在學習的最優策略）」的能力。因為目標策略總是選擇最大 Q
  值，它能保證在無限時間內收斂到馬可夫決策過程（MDP）的絕對最佳策略。但在訓練期間，由於它忽視了正在執行的隨機行為風險，表現可能會非常差。
- **SARSA (On-policy)**：
  其更新基於「實際採取的行動」。它沒有區分行為策略與目標策略，它就是在優化「當下正在執行的
  $\epsilon$-greedy
  策略」。這使得它會忠實反映出探索策略所帶來的風險（例如：走在懸崖邊很容易掉下去），進而學會避開危險區域。

**一般而言**： Q-learning
試圖找出無視探索風險的全局最優解，但在訓練中代價高昂；SARSA
則是務實地找出在當前探索機率下，表現最穩定的妥協策略。

---

## 六、結論要求

總結兩種方法在本實驗中的表現差異：

1. **哪一種方法收斂較快？** 在 $\epsilon$-greedy 策略下，**SARSA
   收斂較快**。它能迅速學會安全路徑並使每回合的累積獎勵趨於穩定，而 Q-learning
   則會因為持續的掉落懸崖而花費更多時間來穩定 Q 值，且平均獎勵表現較差。
2. **哪一種方法較穩定？** **SARSA 較為穩定**。SARSA
   選擇的安全路徑使其不易受到隨機探索動作的影響；Q-learning
   選擇貼著懸崖走，極易因為探索機制的隨機性掉入懸崖，導致單回合獎勵產生劇烈震盪。
3. **在何種情境下應選擇 Q-learning 或 SARSA？**
   - **選擇 SARSA
     的情境**：當「在訓練過程中犯錯的代價非常巨大」時（例如：真實世界的無人機導航、機器人控制，撞毀或損壞需要高昂成本），我們需要演算法在訓練期間也保持安全，此時
     SARSA 的保守特性是最佳選擇。
   - **選擇 Q-learning
     的情境**：當「訓練環境是模擬器且犯錯成本為零」，並且我們最終的目的是要提取出一個純貪婪（Greedy,
     $\epsilon=0$）的絕對最佳策略部署於正式環境時，應選擇
     Q-learning。因為它不受探索策略的限制，能保證找出理論上的最佳解。
